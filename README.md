# H2Odin

A C-header-to-Odin bindings generator, written in Odin.

H2Odin reads C headers with libclang and produces clean, idiomatic Odin bindings â€” configured through a small but powerful Lua policy layer.

> Status: usable pipeline (Milestones 0â€“5 + 6â€“16). The vendor validation corpus (nine examples, including ggml) regenerates and `odin check`s green via `./scripts/validate-examples`. Idiomatic wrappers (`procs.wrappers`) cover out-parameter results and pointer+count input slices per spec 0011.

---

## Features

- **Two type modes.** *ABI mode* preserves the C API faithfully using Odin's C-compatible types (`c.int`, `c.size_t`, â€¦) and never emits wrappers. *Idiomatic mode* uses proven native spellings and will add selected out-parameter-result and pointer+length-slice wrappers over retained faithful foreign declarations. Generic string conversion is not in the initial wrapper set.
- **Measured C bit-fields.** Representable bit-field runs become Odin `bit_field` regions only when their size, alignment, and offsets can be proven from libclang's target layout; everything else fails closed to an opaque record with a diagnostic.
- **Correctness first.** A type is never swapped for a nicer-looking one if it would break behavior or the ABI. When the header is ambiguous, H2Odin picks a safe default, flags it, and lets you override it â€” it never silently guesses wrong.
- **Deterministic.** Same headers plus the same config tree produce identical output, provided config callbacks are themselves deterministic. The Lua config is sandboxed (no `io`/`os`/`debug`, no raw loaders, no `math.random`/`math.randomseed`); `require` only resolves the `h2odin` prelude and sibling `.lua` under the config directory.
- **Configurable in Lua.** Simple libraries need a few lines of data; tricky ones drop into Lua functions for the hard cases â€” same small API either way.
- **Diagnostics report.** Every non-certain decision in a run (guessed pointer lowerings, unresolved idiomatic leaves, opaque layout fallbacks, â€¦) is listed on stderr after generation.

## Requirements

- [Odin](https://odin-lang.org/) compiler (pinned via [mise](https://mise.jdx.dev/) in `.mise/config.toml` as `odin = "dev-2026-07a"`)
- A libclang shared library (the Clang C API), findable by the linker
- `odinfmt` only if you use `./scripts/format`

With [mise](https://mise.jdx.dev/) installed: `mise install` then `mise run <task>`
(or `mise tasks` to list). Tasks are the scripts in `scripts/` discovered via
`#MISE` comment annotations ([file tasks](https://mise.jdx.dev/tasks/file-tasks.html)).
Without mise, install a matching Odin and run those scripts directly.

## Building

```sh
# via mise (recommended â€” uses the pinned Odin toolchain)
mise run check               # type-check the package
mise run build               # produce build/h2odin
mise run test                # unit + e2e tests (builds the binary first)
mise run format              # odinfmt via odinfmt.json
mise run validate-examples   # regen + odin check all validation packages

# same entry points without mise
./scripts/check
./scripts/build
./scripts/test
./scripts/format
./scripts/validate-examples
```

H2Odin is **self-hosted**: the libclang bindings its Extraction stage imports
are generated by H2Odin itself from the pinned headers in
`vendored/libclang/headers/`. `./scripts/regen-libclang` rebuilds the binary
against the checked-in package and regenerates it in place (see
`docs/specs/0002-self-hosted-libclang.md`).

## Usage

```sh
./build/h2odin <project-dir> [options]
./build/h2odin -config:file.lua [options]
```

- **Common case:** pass a project directory. H2Odin loads `H2Odin.lua` from that directory; headers, type mode, naming, output path, and other policy live there (`config.inputs`, `config.type_mode`, `config.output_folder`, â€¦).
- **`-config:path`:** explicit Lua config path when you cannot use the default filename (naming conflicts, shared configs, fixtures).
- **Bindings** are written under `config.output_folder` by default (relative paths resolve against the config directory). Missing `output_folder` is an error â€” not a silent fallback to stdout. Use `-destination:stdout` / `-d:stdout` when you want a single merged unit on stdout.
- **Diagnostics** (warnings and errors) go to stderr only. `-quiet` / `-q` suppresses them (error severities still fail the run). `-verbose` / `-v` adds probable cause and Lua-config fix guidance for each issue.
- `-help` / `-h` prints usage.

Example (writes `examples/sqlite3/sqlite3.odin` via `config.output_folder = "."`):

```sh
./build/h2odin examples/sqlite3
```

Check the full validation corpus (or a single package):

```sh
./scripts/validate-examples
# or one package:
odin check examples/sqlite3 -no-entry-point -collection:vendored=$(pwd)/vendored
```

## Configuration

Configuration is a Lua program that **`require "h2odin"`**, builds a sectioned object with **`h2o.config()`**, and **returns** it. Common cases are plain data; hard cases are callbacks that return a decision, or `nil` to accept the default.

```lua
local h2o = require "h2odin"

local config = h2o.config()
config.package = "raylib"
config.foreign.import_lib = "raylib"
config.type_mode = "idiomatic"

config.naming = h2o.naming.odin {
  strip_prefixes = { proc = "gl", type = "GL", const = "GL_" },
  override = function(sym)
    return sym.default
  end,
}

config.types.overrides = { Vector2 = "[2]f32", Color = "distinct [4]u8" }

config.symbols.remove.where = function(sym)
  return h2o.str.has_prefix(sym.name, "_")
end

return config
```

| Path | Role |
|------|------|
| `package` / `type_mode` | package name; ABI vs idiomatic leaves |
| `inputs` / `preprocess.*` | multi-header inputs; `-I` / `-D` |
| `foreign.import_lib` / `link_prefix` | system library; C symbol prefix |
| `naming.strip_prefixes` | drop a C prefix by kind (`proc` / `type` / `const`) |
| `naming.override` | rename callback |
| `types.map` / `types.overrides` | type spellings (refs only vs also drop the decl) |
| `structs.*` / `procs.*` | field tags/align; param/result spellings and defaults |
| `output.*` / `output_folder` | `layout` (`merged`/`per_header`), footers |
| `symbols.remove.where` | **true drops** a top-level declaration |
| `diagnostics` | per-category severity (`warn` / `error`) |

Unknown keys fail the run with a clear error. Pre-M8 flat keys (`keep`, `rename`, `type_map`, â€¦) are rejected with migration messages.

More detail: [`docs/configuration.md`](docs/configuration.md). Full north-star: [`docs/config-spec.md`](docs/config-spec.md).
